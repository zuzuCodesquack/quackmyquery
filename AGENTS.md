# QuackMyQuery Agent Rules

## Content Polishing Standards
- **Compliance:** Every template page MUST include the Legal Footer with Affiliate and A2P 10DLC disclaimers.
- **Tone:** Use "Expert Guide" tone. Professional but helpful peer-to-peer language.
- **Icons:** Use descriptive emojis for every step in `blueprintSteps` (e.g., 📱 for SMS, 📧 for Email, ⚡ for Trigger).
- **Proofreading:** Always fix typos, ensure consistent capitalization of profession names, and ensure `toolSlug` matches the `make-com` or `zapier` convention.

## Code Standards
- **Next.js:** Use functional components with Tailwind CSS.
- **Copy Logic:** Always include a "Copy to Clipboard" button for the `copyPasteElement`.
- **JSON Integrity:** Never leave trailing commas. Always verify syntax before saving.
- **SEO:** Use semantic HTML (`<main>`, `<header>`, `<h1>`).
## Proactive Problem Solving & Discretion
- **Autonomy:** If you encounter structural errors, logical inconsistencies, deprecated code, or UI bugs that are not explicitly covered in these rules, use your best judgment as a senior engineer to fix them.
- **General Improvements:** You are authorized to make general improvements to code efficiency, Tailwind spacing, accessibility, and UI/UX without asking for permission.
- **Reporting:** If you make a significant discretionary fix that falls outside the explicit rules above, you must briefly summarize what you fixed and why in your final execution report.
